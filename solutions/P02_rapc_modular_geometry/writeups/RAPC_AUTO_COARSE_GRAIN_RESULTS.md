# RAPC automatic coarse-graining toy model results

This is the fifth RAPC falsification gate.

Previous step:

```text
chosen hidden/reference node
-> trace it out
-> effective pair edge
```

This step:

```text
global modular hypergraph
-> scan possible coarse-grainings
-> choose effective/generated pair edge automatically
-> BMV witness
```

The script is:

```text
rapc_auto_coarse_grain_toy.py
```

## Algorithm

Given a global modular seed `K` and state:

```text
rho = exp(-K) / Z
```

the script:

1. Tests every visible pair `(i,j)`.
2. Traces out the complement.
3. Computes:

```text
K_eff(i,j) = -log rho_ij
```

4. Extracts the strongest bilocal Pauli edge from `K_eff`.
5. Compares it to the direct pair coefficient already present in the original
   global `K`.
6. Scores the generated part:

```text
generated = effective - direct
```

This distinction matters:

```text
effective edge = what the observer sees after coarse-graining
generated edge = what coarse-graining created or destroyed
```

## Results

### 1. Local Only

```text
best_effective.visible_pair : None
best_generated.visible_pair : None
```

Interpretation:

Purely local modular data creates no false geometry.

### 2. Direct Pair Only

```text
best_effective.visible_pair : (0, 1)
effective_coeff             : 0.32
direct_coeff                : 0.32
generated_coeff             : ~0
pair_negativity             : 0.298597720681
```

Interpretation:

The effective pair exists, but it was not created by coarse-graining. It was
already present as a direct bilocal term in the global modular Hamiltonian.

So the automatic selector correctly distinguishes:

```text
direct pair geometry != emergent coarse-grained pair geometry
```

### 3. Single Hidden Reference

```text
best_generated.visible_pair : (0, 1)
traced_out                  : (2, 3)
axis                        : ('Z', 'Z')
effective_coeff             : -0.177419370572
direct_coeff                : ~0
generated_coeff             : -0.177419370572
pair_negativity             : 0.173719578459
expected_concurrence        : 0.347439156917
```

Interpretation:

The algorithm finds the pair whose geometry is generated by tracing out the
hidden/reference structure. The observer sees an effective `Z0 Z1` relation even
though the global modular Hamiltonian had no direct `Z0 Z1` term.

### 4. Competing Hidden References

```text
best_generated.visible_pair : (1, 2)
traced_out                  : (0, 3)
axis                        : ('X', 'X')
effective_coeff             : -0.223790034167
direct_coeff                : ~0
generated_coeff             : -0.223790034167
pair_negativity             : 0.216392621094
expected_concurrence        : 0.432785242188
```

Interpretation:

When multiple hidden-reference mechanisms exist, the selector chooses the
stronger generated edge. In this case the emergent `XX` relation beats the
emergent `ZZ` relation.

### 5. Mixed Direct And Generated

```text
best_generated.visible_pair : (0, 1)
axis                        : ('Z', 'Z')
effective_coeff             : 0.0725806294278
direct_coeff                : 0.25
generated_coeff             : -0.177419370572
pair_negativity             : 0.0723259972105
expected_concurrence        : 0.144651994421
```

Interpretation:

This is the most useful result. Coarse-graining does not merely create edges; it
can renormalize them, including destructive renormalization:

```text
direct edge 0.25
generated correction -0.177419370572
effective edge 0.0725806294278
```

So RAPC naturally acquires a Wilsonian flavor:

```text
microscopic modular hypergraph
-> coarse-graining
-> renormalized pair geometry
```

## What RAPC Gains

The chain now looks like:

```text
global modular state
-> automatic scan of coarse-grainings
-> selected observer-visible pair graph
-> BMV witness
```

This is the first prototype where the coarse-graining map is not fully chosen by
hand. It is selected by a variational-style rule:

```text
maximize generated bilocal modular coupling
```

That is not yet a physical principle, but it is a candidate mathematical
selection rule.

## What This Suggests

The emergent geometry in RAPC should probably be defined as a fixed point or
stable structure of repeated maps:

```text
K_micro
-> C_lambda(K_micro)
-> K_eff
-> C_lambda(K_eff)
-> ...
```

where each step selects observer-accessible bilocal structure from hidden
hypergraph correlations.

This turns the next problem into a renormalization problem:

```text
Do repeated RAPC coarse-grainings flow toward stable low-dimensional pair graphs-
```

That is much closer to the original goal:

```text
hypergraph correlations -> emergent local geometry
```

## Remaining Imports

The model still assumes:

```text
finite Type I tensor factors
thermal modular state exp(-K)/Z
partial trace as the coarse-graining operation
Pauli basis
selection by generated pair strength
modular time parameter
```

The next hard step is therefore:

```text
iterate the automatic coarse-graining
and test whether stable effective graphs appear
```

If the stable graphs are arbitrary, RAPC has no geometric selection principle.
If they flow toward sparse/local/low-dimensional graphs, we have the first real
hint of emergent geometry.
